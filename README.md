## Philosophy

```rust
fn approach() -> &'static str {
    // Ship fast. Stay correct. Own the stack.
    "If it's in production, it should be bulletproof."
}
```

I default to Rust for anything where correctness, throughput, or latency is non-negotiable. I've seen enough Python bots and Node.js scrapers crumble under load to know the difference. That said, I pick the right tool вҖ” TypeScript on the frontend, Solidity for contracts, Python for quick data analysis.

I care about: zero-downtime deployments, structured logging, proper error handling, and code that the next engineer (or future me at 2am) can actually read.
